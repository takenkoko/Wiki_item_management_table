# Wiki_item_management_table

複数のJSONデータからアイテム情報を抽出し、重複なしでGoogleスプレッドシートへ自動同期・進捗の視覚化を行うPythonスクリプトです。
ゲーム（アークナイツ：エンドフィールド等）のWiki編集チームにおける進捗管理の自動化を目的として開発しました。

##  主な機能・特徴
- **大量データの一括統合**: 9つの異なるJSONファイルを巡回し、重複なしで1,000件超のデータを一瞬で抽出・結合します。
- **進捗の自動色分け**: 各アイテムのステータス（「完了」/「未完了」）を判定し、スプレッドシートの行全体を自動でカラーリング（緑/赤）します。
- **リアルタイム集計表**: データの同期と同時に、シートの右側に現在の完了数・未完了数・合計件数の集計表を自動生成します。
- **API速度制限（Quota Exceeded）対策**: Google APIへのリクエスト過多によるエラー（429や400）を回避するため、`batch_updater` を使用した一括リクエスト（バルク処理）構造へ最適化。動作の軽量化と高速化を実現しています。
- **Discord通知連携**: 処理完了時に、進捗ステータスとシートの確認URLを指定のDiscordチャンネルへ自動で通知します。
