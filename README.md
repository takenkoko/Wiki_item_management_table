# Wiki_item_management_table(ver.02)

開発成果

- 9種類のJSONファイルを統合
- 1,000件以上のデータを自動管理
- Google Sheets APIとの自動同期を実現
- Batch UpdateによるAPI通信回数の最適化
- Discord通知による運用自動化

複数のJSONデータからアイテム情報を抽出し、重複なしでGoogleスプレッドシートへ自動同期・進捗の視覚化を行うPythonツールです。
アークナイツ：エンドフィールドWiki編集では複数のJSONファイルを個別に確認する必要があり、編集状況の把握に時間がかかっていました。この課題を解決するため、データ統合・進捗管理・通知を自動化するPythonツールを開発しました。

##  主な機能・特徴
* **大量データの一括統合:** 9つの異なるJSONファイルを巡回し、重複なしで1,000件超のデータを一瞬で抽出・結合します。
* **進捗の自動色分け:** 各アイテムのステータス（「完了」/「未完了」）を判定し、スプレッドシートの行全体を自動でカラーリング（緑/赤）します。
* **リアルタイム集計表:** データの同期と同時に、シートの右側に現在の完了数・未完了数・合計件数の集計表を自動生成します。
* **シートデザインの自動最適化:** シート全体の統一感を高めるため、ワークシートタブのカラーカスタマイズ（ダークグリーン化）や、右側の集計表の横幅（G列〜H列）にぴったり美しく収まる利用説明文（ガイドライン）の自動生成・装飾を同時に実行します。
* **API速度制限（Quota Exceeded）およびエラー対策:** Google APIへのリクエスト過多によるエラー（429や400）を回避するため、`batch_updater` を使用した一括リクエスト（バルク処理）構造へ最適化。さらに、各セルの装飾リクエストには型安全な `CellFormat` オブジェクトを徹底することで、Google APIとの通信エラーを排除し、高い動作軽量化・高速化・堅牢性を実現しています。
* **Discord通知連携:** 処理完了時に、最新の進捗ステータス（各件数）とシートの確認URLを指定のDiscordチャンネルへ自動で通知します。
